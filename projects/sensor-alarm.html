Automated Sensor & Security Alarm System

Hardware: STM32F103VET6, SR602 PIR, 4x3 Keypad | Language: C++ / Bare-Metal Hardware HAL

This project involved developing a real-time bare-metal security control node centered around the ARM Cortex-M3 architecture. Utilizing the STM32F103VET6 MCU, the objective was to build a highly responsive system capable of low-power motion telemetry tracking alongside synchronous passcode authorization matrices.

1. Firmware Architecture & Peripheral Integration

The system logic operates on a strict Finite State Machine (FSM) architecture integrated into a deterministic timer-driven polling schedule. The SR602 pyroelectric infrared (PIR) motion sensor is assigned to an external interrupt line (EXTI) configured for rising-edge triggers to ensure immediate, microsecond-level latency signaling upon detection. Concurrently, the 4x3 matrix keypad utilizes a dynamic row-scanning technique combined with configured internal pull-up GPIO registers to parse inputs without halting core system loops.

2. Engineering Difficulties & Optimization Strategies

  • Mechanical Switch Debouncing & Race Conditions: A major challenge arose from mechanical noise during matrix keypad decoding, which triggered ghost inputs. This was resolved by designing an asynchronous firmware debounce filter using localized hardware timer tickers rather than basic execution blocking (`HAL_Delay`).
  • Power Consumption vs. Latency Trade-offs: Maintaining high responsiveness for the SR602 sensor while scanning the matrix keypad risked high current draw. To optimize efficiency, the MCU core was configured to enter low-power Sleep mode, waking up dynamically via EXTI interrupt lines only when hardware activity is detected.
  • Priority Inversion on the Nested Vector Interrupt Controller (NVIC): High-frequency keypad scanning routines occasionally clashed with critical sensor alert sequences. This required careful configuration of sub-priority grouping inside the NVIC register maps to ensure real-world threat indicators reliably take precedence over operational inputs.
